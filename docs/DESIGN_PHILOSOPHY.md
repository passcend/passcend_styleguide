# Passcend UI/UX 디자인 철학

> 사용자 친화적 디자인 원칙

---

## 🎯 핵심 철학

### 1. 대화형 타이틀

딱딱한 폼 레이블 대신 사용자에게 말을 건네는 듯한 헤더 사용

| ❌ Before | ✅ After                     |
| --------- | ---------------------------- |
| Password  | 비밀번호를 입력해주세요      |
| Email     | 이메일 주소가 어떻게 되세요? |
| Error     | 앗, 뭔가 잘못됐어요          |

### 2. 중요도 기반 배치

가장 중요한 요소를 크게 배치하여 사용자 집중도 향상

- 주요 액션 버튼: 크고 눈에 띄게
- 보조 정보: 작고 은은하게

### 3. 직관적인 UI

별도 설명 없이도 이해 가능한 인터페이스

- 아이콘과 텍스트의 조화
- 예측 가능한 동작

### 4. 친절한 안내

에러 메시지도 친근하고 해결책 제시

- "다시 시도해볼까요?"
- "이런 방법도 있어요"

### 5. 마이크로 인터랙션

버튼 클릭, 체크박스, 입력 필드 등 모든 요소에 부드러운 애니메이션

- 호버 효과
- 클릭 피드백
- 로딩 상태

---

## 🧭 플로우 원칙

### 6. 단일 책임 화면

한 화면에 하나의 목적만 담기

```
이메일 입력 → 비밀번호 입력 → 완료
(각각 별도 화면)
```

### 7. 진행 상태 명시

사용자가 현재 위치와 남은 단계를 알 수 있도록

- 스텝 인디케이터
- 프로그레스 바

### 8. 되돌리기 가능

실수해도 쉽게 뒤로 갈 수 있는 안심감

- 명확한 뒤로가기 버튼
- 취소 확인 다이얼로그

---

## 📐 레이아웃 원칙

### 9. 여백의 미학

빽빽한 폼이 아닌 넉넉한 공간으로 시각적 호흡

- 충분한 패딩
- 섹션 간 구분

---

## ⚡ 피드백 원칙

### 10. 즉각적인 피드백

입력 즉시 유효성 검사, 실시간 응답

- 타이핑하는 순간 검증
- 성공/실패 즉시 표시

### 11. 맥락적 도움말

물음표 버튼 대신 입력 필드 바로 아래에 힌트 텍스트

### 12. 감정적 연결

성공 시 축하 애니메이션 🎉, 격려 메시지

- "완료됐어요!"
- "잘하고 있어요"

---

## 🔮 예측 원칙

### 13. 의도 예측

사용자의 다음 행동을 예측해서 미리 준비

- 이메일 필드: 이메일 키보드
- 숫자 필드: 숫자 키패드
- 자동 포커스 이동

---

## 📁 적용 우선순위

1. **Setup 플로우** (회원가입) - 첫인상
2. **로그인** - 매일 사용
3. **비밀번호 추가/수정** - 핵심 기능
4. **메인 대시보드** - 일상 사용
5. **설정 화면** - 고급 사용자
